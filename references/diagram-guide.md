# 图表类型匹配指南

本文件指导如何根据内容的逻辑结构选择合适的图表类型，
以及如何使用 HTML/CSS/SVG 绘制这些图表。

---

## 匹配决策树

分析内容时，按以下逻辑判断：

1. **内容描述的是有先后顺序的步骤/阶段？** → 流程图
2. **内容呈现层级关系（上下级、父子、分类）？** → 树状图/结构图
3. **内容描述战略框架（基础-支柱-目标）？** → 信息屋/架构图
4. **内容在做对比分析（A vs B、优劣势）？** → 对比表/象限图
5. **内容展示多维度能力或占比分布？** → 雷达图/饼图
6. **内容是并列的几个要点，无明显逻辑关系？** → 卡片式图标列表（不需要复杂图表）

**注意**：一个 OnePage 中可以有 1-3 个不同类型的图表，但不要超过 3 个，否则视觉会混乱。

## 图表配色规范

图表配色必须与当前设计风格一致。以下是各风格的图表配色速查：

### B1 暗夜编辑
- 主数据色：`#f59e0b` 琥珀
- 次要数据色：`#14b8a6` 青绿
- 对比数据色：`#f87171` 柔和红
- 网格/轴线：`#1e293b` 深灰
- 背景：透明或 `#111118`

### B2 瑞士精密
- 主数据色：`#dc2626` 正红
- 次要数据色：`#000000` 纯黑（不同透明度）
- 对比数据色：`#71717a` 中灰
- 网格/轴线：`#000000` 纯黑
- 背景：透明

### B3 有机自然
- 主数据色：`#c2410c` 陶土
- 次要数据色：`#4d7c0f` 苔藓绿
- 对比数据色：`#a8a29e` 浅灰
- 网格/轴线：`#e7e5e4` 暖灰白
- 背景：透明或 `#f5f0e8`

### B4 建筑粗野
- 主数据色：`#ea580c` 安全橙
- 次要数据色：`#facc15` 电黄
- 对比数据色：`#000000` 纯黑
- 网格/轴线：`#000000` 纯黑（粗线 2px）
- 背景：`#e5e5e5`

### B5 新波普编辑
- 主数据色：`#e11d48` 亮粉
- 次要数据色：`#2563eb` 宝蓝
- 对比数据色：`#fbbf24` 明黄
- 网格/轴线：`#000000` 纯黑
- 背景：对应色块的浅色版

### B6 极简东方
- 主数据色：`#1c1917` 墨黑
- 次要数据色：`#b91c1c` 朱红（极少使用）
- 对比数据色：`#a8a29e` 浅灰
- 网格/轴线：`#e7e5e4` 极浅灰（0.5px）
- 背景：透明

### B7 数据新闻
- 主数据色：`#1e3a5f` 深海军蓝
- 次要数据色：`#9f1239` 锈红
- 对比数据色：`#78716c` 中灰
- 网格/轴线：`#d6d3d1` 暖灰（1px）
- 背景：透明

### B8 赛博街道
- 主数据色：`#ff006e` 霓虹粉
- 次要数据色：`#00f5d4` 电青
- 对比数据色：`#737373` 中灰
- 网格/轴线：`#262626` 深灰
- 背景：透明或 `#111111`

**禁止**：在任何图表中使用蓝紫渐变（如 `#6366f1` → `#a855f7`）。

## 图表绘制规范

### 通用规则
- 所有图表必须使用 CSS/SVG 绘制，不依赖外部图片
- 图表配色必须与当前设计风格一致（使用上方表格中的颜色）
- 图表中的文字使用与正文相同的字体栈
- 图表需要有明确的标题
- 在容器内预留适当的内边距
- 数据标签使用等宽字体（B1/B7/B8）或与正文一致的字体（其他风格）

### 流程图 (Flowchart)

使用 CSS Flexbox/Grid 布局 + SVG 箭头连接。

结构示例：
```
[步骤1] → [步骤2] → [步骤3] → [步骤4]
```

实现要点：
- 每个步骤使用一个矩形卡片（圆角根据风格：B2=0, B3=20px, 其他=8px）
- 纵向长图中可使用纵向排列（上→下）
- 横向宽图中使用横向排列（左→右）
- 箭头使用 SVG `<line>` + `<polygon>` 箭头头部，或 CSS `::after` 伪元素
- 步骤卡片内包含：步骤编号 + 标题 + 简短描述（可选）
- **禁止使用负 margin 或 magic number 定位**（如 `top: -36px`）

### 树状图/结构图 (Tree/Structure)

使用 CSS 嵌套列表 + 连接线。

结构示例：
```
        [根节点]
       /    |    \
   [子1]  [子2]  [子3]
   / \
[叶1][叶2]
```

实现要点：
- 使用 CSS `border-left` + `::before` 伪元素绘制树线
- 或使用 SVG `<path>` 绘制曲线连接
- 每层缩进 24-32px
- 节点样式使用当前风格的卡片样式

### 信息屋/架构图 (Architecture/House)

模拟房屋结构：屋顶(目标) → 支柱(核心能力) → 地基(基础设施)。

结构示例：
```
╔═══════════════════════════╗
║       顶层目标/愿景         ║  ← 屋顶
╠════╦════╦════╦════════════╣
║支柱║支柱║支柱║  支柱       ║  ← 柱子（核心模块）
║ 1  ║ 2  ║ 3  ║   4        ║
╠════╩════╩════╩════════════╣
║       底层基础/平台         ║  ← 地基
╚═══════════════════════════╝
```

实现要点：
- 使用 CSS Grid 三行布局
- 屋顶和地基：横跨全宽
- 中间支柱：等宽列
- 每部分使用不同深浅的主题色填充（**纯色，非渐变**）

### 对比图/象限图 (Matrix/Quadrant)

使用 CSS Grid 2×2 布局。

结构示例：
```
          高
     ┌────┬────┐
     │ Q2 │ Q1 │
     ├────┼────┤
     │ Q3 │ Q4 │
     └────┴────┘
  低          高
```

实现要点：
- 2×2 Grid，每格不同背景色（使用主题色的纯色或不同透明度）
- 轴标签使用绝对定位或 Grid 额外行/列
- 数据点使用彩色圆点标注
- **拒绝使用渐变填充象限**，使用纯色或透明度变化

### 雷达图/饼图 (Radar/Pie)

使用内联 SVG 绘制。

实现要点：
- **饼图**：使用 SVG `<circle>` + `stroke-dasharray` 技巧
- **雷达图**：使用 SVG `<polygon>` 绘制多边形
- 颜色使用当前风格的强调色系列（**纯色 stroke/fill，非渐变**）
- 添加图例说明
- 控制尺寸在 200-300px 范围内，避免过大
- 数据标签使用与正文相同的字体栈

### 柱状图/条形图 (Bar Chart)

使用 CSS Flexbox 或 SVG 绘制。

实现要点：
- 柱状图：flex-column，高度由数据值决定
- 条形图：flex-row，宽度由数据值决定
- 柱子使用纯色填充（当前风格的主数据色 + 次要数据色交替）
- 添加数值标签在柱子顶部或内部
- 坐标轴使用细线（颜色参考上方各风格的"网格/轴线"色）
- **绝对禁止渐变柱子**

### 折线图 (Line Chart)

使用 SVG `<polyline>` 或 `<path>` 绘制。

实现要点：
- 线条使用 2-3px 纯色 stroke
- 数据点使用圆点标记
- 填充区域可使用极低透明度的纯色（如 `rgba(245,158,11,0.1)`）
- 坐标轴和网格线参考各风格规范
- **线条必须是纯色，禁止渐变描边**
