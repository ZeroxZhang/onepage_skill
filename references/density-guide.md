# 信息密度与内容改写指南

本文件定义了三种信息密度模式下，内容应如何被改写和排版。

---

## 跨密度内容原则

无论选择哪种密度，改写时都必须遵循以下内容原则：

1. **结论先行**：标题或首屏即给出核心论点，不把结论藏到最后
2. **So What 测试**：改写后对每个保留的句子追问"所以呢？这对读者意味着什么？"，删除或重写无法通过的句子
3. **量化替换**：将模糊形容词替换为具体数字。若素材未提供数字，可基于行业常识标注近似值并注明"[估算]"
4. **MECE 校验**：确保改写后的模块间不重叠、不遗漏
5. **类型适配**：根据内容类型（T1-T5）确保必备要素不被删除（参考 type-differentiation.md）

---

## C1: 低密度 (Low Density)

### 内容改写规则
- **大幅精简**：每个内容模块只保留 1 个核心金句或关键词组
- 删除所有详细解释、案例、数据佐证
- 将段落压缩为一句话
- 如果原文有 5 个以上模块，合并或删减至 3-4 个

### 排版规则
- 大面积留白：内容区域占画布面积不超过 50%
- 字号偏大：标题 40-56px，正文 18-20px
- 图表简化：仅保留最核心的一个图表
- 模块间距加大：48-64px
- 优先使用图标/数字大卡片替代文字段落

### 视觉比例
```
留白 : 文字 : 图表 = 5 : 3 : 2
```

### 类型改写提示
- **T1 产品介绍**：价值主张保留为独立首屏金句，功能点压缩为关键词
- **T2 产品说明**：保留核心步骤的编号，参数合并为简洁规格表
- **T3 方案介绍**：方法论提炼为一句话，保留 1 个最强案例数据
- **T4 操作指南**：保留关键步骤编号，删减详细说明仅留动作短语
- **T5 信息图**：保留最核心的 1 组数据对比，图表简化为关键数字

---

## C2: 中密度 (Medium Density)

### 内容改写规则
- **结构化优化**：为每个模块提炼 4-6 字的小标题
- 每个模块保留 2-3 句核心说明
- 保留关键数据和论据，删除冗余表述
- 可使用项目符号列表提升信息扫描效率
- 原文模块数量可保持不变

### 排版规则
- 均衡布局：内容区域占画布面积约 65-75%
- 标准字号：标题 28-36px，正文 15-16px
- 图文配合：保留 1-2 个图表，与文字交替排列
- 模块间距：32-40px
- 可使用小标题 + 分隔线组织信息

### 视觉比例
```
留白 : 文字 : 图表 = 3 : 4 : 3
```

### 类型改写提示
- **T1 产品介绍**：价值主张保留为独立首屏，功能点压缩为关键词 + 1 句话说明
- **T2 产品说明**：保留完整步骤编号，注意事项合并为要点列表
- **T3 方案介绍**：保留方法论框架和 1-2 个关键案例数据
- **T4 操作指南**：保留步骤编号 + 简短说明，关键提示用强调色标注
- **T5 信息图**：保留 2 组核心数据对比，配 1-2 个图表

---

## C3: 高密度 (High Density)

### 内容改写规则
- **完整保留或适度扩写**：保留原文的全部信息
- 如果原文信息不够丰富，可以合理扩写（添加解释、逻辑连接）
- 使用小标题、编号列表、加粗关键词提升结构化程度
- 不删减任何有价值的信息

### 排版规则
- 信息密集：内容区域占画布面积 80-90%
- 多栏布局：纵向长图使用双栏，横向宽图使用三栏
- 字号紧凑：标题 24-30px，正文 14-15px
- 卡片式布局：每个模块用卡片包裹，减少视觉混乱
- 可使用 2-3 个图表
- 模块间距收紧：20-28px
- 纵向长图高度会更长

### 视觉比例
```
留白 : 文字 : 图表 = 1.5 : 5 : 3.5
```

### 类型改写提示
- **T1 产品介绍**：完整保留功能点说明 + 使用场景 + CTA，可适度扩写痛点描述
- **T2 产品说明**：完整保留步骤流程 + 参数规格 + 注意事项，多栏/卡片消化密集参数
- **T3 方案介绍**：完整保留方法论 + 多案例数据 + 交付物 + 团队背书
- **T4 操作指南**：完整保留编号步骤 + 关键提示 + FAQ，使用卡片分区
- **T5 信息图**：完整保留数据叙事线 + 多组对比 + 来源标注，2-3 个图表

---

## 跨密度通用规则

1. 无论哪种密度，都必须保留原文的**核心观点**，不可曲解
2. 改写时保持**专业性**，不降低语言质量
3. 如果用户的原文已经非常精炼，低密度模式下不要过度删减至失去意义
4. 如果用户的原文非常简短，高密度模式下应基于主题合理扩写
5. 每次改写后，务必核对：改写后的文案是否完整传达了原文意图？是否有任何信息被遗漏？是否有任何信息被错误理解？
6. **So What 终检**：改写完成后，对每个保留的句子追问"所以呢？这对读者意味着什么？"，删除或重写无法通过的句子
