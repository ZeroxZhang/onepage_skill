# 尺寸与版式规范

本文件定义了三种尺寸模式的完整版式规范。

---

## A1: 纵向长图 (Portrait / Scroll-Friendly)

### 画布参数
- 宽度：800px（固定）
- 高度：根据内容自适应（不设上限，通常 1200-3000px）
- 外边距（页面 padding）：48px 左右，40px 上下

### 版式结构（从上到下）
```
┌──────────────────────────────────┐
│          顶部区域 (Header)         │  ← 主标题 + 副标题 + 元信息
│  渐变/装饰背景条                    │
├──────────────────────────────────┤
│          核心摘要区                 │  ← 1-2 句话概述（可选）
├──────────────────────────────────┤
│          内容模块 1                 │  ← 小标题 + 正文/图表
├──────────────────────────────────┤
│          内容模块 2                 │
├──────────────────────────────────┤
│          内容模块 3                 │
├──────────────────────────────────┤
│          ...更多模块...             │
├──────────────────────────────────┤
│          底部区域 (Footer)          │  ← 来源/署名/日期
└──────────────────────────────────┘
```

### 排版特点
- 单栏为主，信息从上到下线性流动
- 高密度模式下可使用双栏并排卡片
- 模块间距：32-48px
- 图表宽度可占满内容区或 80% 居中

### 字号规范
> 参考 `typography.md` 中的尺寸-字号对照表

- **正文**：15-16px（约画布宽度的 1.9%-2.0%）
- **主标题**：32-48px
- **副标题**：20-24px
- **辅助说明**：12-13px

---

## A2: 横向宽图 (Landscape / Presentation)

### 画布参数
- 宽度：1920px
- 高度：1080px（固定）
- 外边距：64px 左右，48px 上下

### 版式结构
```
┌─────────────────────────────────────────────────────┐
│  左侧标题区 (30%)   │   右侧内容区 (70%)              │
│                     │   ┌────────┐ ┌────────┐        │
│  主标题             │   │ 模块 1  │ │ 模块 2  │        │
│  副标题             │   └────────┘ └────────┘        │
│  摘要               │   ┌────────┐ ┌────────┐        │
│                     │   │ 模块 3  │ │ 图表    │        │
│                     │   └────────┘ └────────┘        │
└─────────────────────────────────────────────────────┘
```

**备选布局**（根据内容选择）：
- **顶部标题 + 下方多栏**：标题占顶部 20%，内容区使用 2-3 栏
- **左右分栏**：左侧 30% 为导航/标题，右侧 70% 为内容

### 排版特点
- 必须使用多栏布局（2-3 栏），禁止单栏
- 内容需高度紧凑以适应固定高度
- 模块间距：24-32px

### 字号规范
> 参考 `typography.md` 中的尺寸-字号对照表

- **正文**：18-22px（约画布宽度的 1.0%-1.1%，注意：A2 画布宽是 A1 的 2.4 倍，字号需相应放大）
- **主标题**：48-72px
- **副标题**：28-36px
- **小标题**：22-28px
- **辅助说明**：14-16px

**⚠️ 常见错误**：A2 画布更大，字号应该放大而非缩小。切勿使用 A1 的字号范围（15-16px 正文），否则会导致大面积留白。

### ⚠️ 固定高度布局规则（A2 必读）

A2 的画布高度是 **固定的 1080px**，与 A1（高度自适应）有本质区别。生成 CSS 时必须遵循以下规则，防止内容与容器之间出现不可控的空白：

1. **先计算垂直空间预算，再写 CSS**：
   - 可用高度 = 1080px - 上下 padding (96px) = **984px**
   - 将 984px 按 header / 内容行1 / 内容行2 / ... / footer 分配具体像素值
   - 将计算结果写入 `grid-template-rows`（如 `80px 1fr 120px 1fr 40px`）
   - 确保各行高度之和 ≈ 可用高度的 **90-100%**

2. **Grid 容器必须显式声明 `grid-template-rows`**：
   - 为每一行分配明确的高度比例或固定值
   - **禁止**依赖 grid 默认的 `stretch` 行为自动分配行高
   - 同时必须设置 `align-content: start`，确保内容从顶部开始排列
   ```css
   /* ✅ 正确示例 */
   .main-content {
       display: grid;
       grid-template-columns: 1fr 1fr 1fr;
       grid-template-rows: 280px 100px 280px;  /* 明确的行高 */
       align-content: start;
       gap: 16px;
   }
   
   /* ❌ 错误示例 — 会导致大面积空白 */
   .main-content {
       display: grid;
       grid-template-columns: 1fr 1fr 1fr;
       /* 未声明 grid-template-rows，默认 stretch 拉伸 */
       flex: 1;  /* 危险：撑满剩余空间后行会被过度拉高 */
   }
   ```

3. **禁止使用 `flex: 1` 撑满剩余空间**（除非配合明确的子元素高度约束）：
   - 在固定高度容器中，`flex: 1` 会让某个区域获得远超内容的高度
   - Grid/Flex 子元素在高度远小于容器时，会产生大面积不可控空白

4. **Flex 纵向容器必须使用 `justify-content` 控制分布方式**：
   - 推荐 `justify-content: flex-start` 或 `space-between`
   - **禁止**使用默认值（等效于 `stretch`/`normal`），否则内容可能被推到中部或底部

5. **内容总高度应填满画布的 85-95%**：
   - 如果内容不足以填满，应通过增大 padding、扩大图表区域、增加装饰元素等方式补足
   - 绝不能让布局引擎自动分配空白

### ⚠️ 多栏底部对齐规则

当使用左右分栏布局时，两侧底部元素的垂直位置必须一致：

**❌ 错误示例 — 两侧策略不一致：**
```css
/* 左侧用 flex + margin-top: auto */
.left-panel {
    display: flex;
    flex-direction: column;
}
.left-bottom { margin-top: auto; }  /* 推到底部 */

/* 右侧用固定行高 */
.right-panel {
    display: grid;
    grid-template-rows: 280px 200px 240px 60px;  /* 60px 固定位置 */
}
```
→ 结果：两侧底部元素不对齐

**✅ 正确示例 — 两侧策略一致：**
```css
/* 左侧 */
.left-panel {
    display: grid;
    grid-template-rows: auto auto 1fr;  /* 1fr 推到底部 */
}

/* 右侧 */
.right-panel {
    display: grid;
    grid-template-rows: 280px 200px 240px 1fr;  /* 同样用 1fr */
}
```
→ 结果：两侧底部元素水平对齐

### ⚠️ 视觉重心平衡规则

左右分栏布局时，两侧的视觉重量必须均衡，避免一侧"头重脚轻"或"空洞无物"。

**核心原则：**
1. **内容密度平衡**：左右两侧的内容填充度应相近，避免一侧密集、一侧稀疏
2. **色彩重量平衡**：深色/高饱和色块视觉重量大，需与另一侧的丰富内容形成平衡
3. **层次结构平衡**：两侧都应有明确的视觉层次（标题→副标题→正文→辅助）

**检查方法：眯眼测试**
- 眯起眼睛看整体画面，左右两侧的"视觉重量感"应大致相等
- 如果一侧明显更"重"或更"空"，需要调整

**❌ 错误示例 — 视觉重心失衡：**
```
左侧：大标题 + 简短描述 + 小卡片（视觉轻）
右侧：6个彩色大卡片 + 多行文字（视觉重）
```
→ 结果：右侧压倒左侧，画面倾斜

**✅ 正确示例 — 视觉重心平衡：**
```
左侧：品牌色块（深色，视觉重）+ 多个色块卡片（内容丰富）
右侧：6个彩色特性卡片 + 安装卡片 + 文件结构（内容丰富）
```
→ 结果：左侧有色彩重量，右侧有内容数量，整体平衡

**调整策略：**
- 若左侧偏轻：增加色彩对比强烈的元素（如彩色背景卡片、强调色块）
- 若右侧偏轻：增加内容模块数量或使用更丰富的布局结构
- 可通过左右两侧的布局结构调整（如左侧用垂直堆叠，右侧用网格矩阵）来创造视觉平衡

### ⚠️ 绝对定位与层叠规则

A2/A3 中使用绝对定位极易导致元素错层、遮挡等问题，必须严格遵循以下规则：

**1. 禁止对内容元素使用绝对定位**：
- 所有内容元素（包括 header、正文模块、footer）必须在 Grid 布局内
- 绝对定位仅可用于纯装饰元素（背景图形、装饰线条、几何点缀等）
- 装饰元素不得承载任何需要阅读的信息

**2. Footer 必须纳入 Grid 布局**：
- Footer 应作为 grid 的最后一行（使用 `1fr` 或固定高度）
- 禁止将 footer 作为独立元素绝对定位在画布底部
- 绝对定位的 footer 会遮挡上方内容区域

**3. 装饰元素的层叠规范**：
- 装饰元素必须设置 `pointer-events: none`
- 装饰元素的 `z-index` 应设为 `0` 或更低（负值）
- 内容区域的 `z-index` 应高于装饰元素

**❌ 错误示例 — 内容元素使用绝对定位：**
```css
.main-container {
    height: 100%;  /* 占满画布 */
}
.footer {
    position: absolute;
    bottom: 24px;  /* 绝对定位在底部 */
    z-index: auto; /* 未设置，会遮挡 .main-container */
}
```
→ 结果：footer 叠加在 .main-container 上方，遮挡底部内容

**✅ 正确示例 — 所有内容在 Grid 内：**
```css
.main-container {
    display: grid;
    grid-template-rows: auto 1fr auto;  /* footer 作为最后一行 */
}
.footer {
    /* 无需绝对定位，自动在 grid 底部 */
}
.decoration {
    position: absolute;
    pointer-events: none;
    z-index: 0;  /* 低于内容区域 */
}
```
→ 结果：布局清晰，无遮挡问题

## A3: 正方形 (Square / Social Media)

### 画布参数
- 宽度：1080px
- 高度：1080px（固定）
- 外边距：48px

### 版式结构
```
┌───────────────────────────────┐
│         主标题区               │  ← 上方 15-20%
├───────────────────────────────┤
│    ┌─────────┐ ┌─────────┐   │
│    │ 模块 1   │ │ 模块 2   │   │  ← 中间 60-70%
│    └─────────┘ └─────────┘   │    2×2 或 2×3 网格
│    ┌─────────┐ ┌─────────┐   │
│    │ 模块 3   │ │ 模块 4   │   │
│    └─────────┘ └─────────┘   │
├───────────────────────────────┤
│         底部信息               │  ← 下方 10-15%
└───────────────────────────────┘
```

### 排版特点
- 空间极其宝贵，优先精简内容
- 建议使用 2×2 或类似的网格布局
- 图表需要缩放至适合单个网格格子

### 字号规范
> 参考 `typography.md` 中的尺寸-字号对照表

- **正文**：16-18px（约画布宽度的 1.5%-1.7%）
- **主标题**：36-52px
- **副标题**：22-28px
- **小标题**：18-24px
- **辅助说明**：12-14px

**注意**：A3 正方形空间紧张，字号取中间值，确保内容紧凑但可读。

### ⚠️ 固定高度布局规则（A3 必读）

A3 与 A2 一样是固定高度画布（1080×1080px），**必须遵循与 A2 相同的固定高度布局规则**（见上方 A2 章节的「固定高度布局规则」）。

关键差异：
- 可用高度 = 1080px - 上下 padding (96px) = **984px**
- 正方形空间更紧张，垂直空间预算分配需要更精细
- 建议使用 2×2 Grid 并显式声明 `grid-template-rows`，每行约 400-450px
